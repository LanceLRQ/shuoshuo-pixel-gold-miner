# 黄金矿工 H5 - 执行计划

## 总览

项目分为 6 个阶段，按依赖关系顺序推进。每个阶段有明确的交付物和验收标准。

---

## Phase 1 - 基础框架

**目标：** 搭建项目骨架，实现渲染和输入系统

### 任务清单

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|---------|
| 1.1 | 项目初始化 | Vite + TypeScript 项目配置 | `npm run dev` 可启动，`npm run build` 可构建 |
| 1.2 | Canvas 渲染器 | `core/Renderer.ts` | Canvas 正确创建，自适应缩放，`imageSmoothingEnabled = false` |
| 1.3 | 游戏主循环 | `main.ts` + `core/Game.ts` | requestAnimationFrame 驱动，deltaTime 正确计算 |
| 1.4 | 输入系统 | `core/Input.ts` | 键盘/鼠标/触摸事件统一封装，可检测按键状态 |
| 1.5 | 场景管理器 | `scene/SceneBase.ts` + 场景切换逻辑 | 场景可注册、切换，生命周期方法正确调用 |

### 依赖关系

```
1.1 → 1.2 → 1.3
1.1 → 1.4
1.3 → 1.5
```

### 阶段验收

- 空白 Canvas 正确显示并自适应窗口大小
- 控制台能输出帧率和输入事件
- 场景切换机制可演示

---

## Phase 2 - 像素素材系统

**目标：** 建立纯代码像素素材体系

### 任务清单

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|---------|
| 2.1 | PixelMap 类型与渲染 | 类型定义 + `drawPixelMap()` | 能将 PixelMap 数据正确渲染到 Canvas |
| 2.2 | 预渲染缓存系统 | `createSpriteCache()` | 精灵预渲染到离屏 Canvas，后续用 drawImage 绘制 |
| 2.3 | 核心精灵数据 | `assets/sprites.ts` 中角色/矿物精灵 | 矿工、钩爪、各类矿物的像素数据完成 |
| 2.4 | 背景渲染 | 天空/地表/地下的渲染逻辑 | 三层背景正确显示，配色符合设计 |
| 2.5 | 像素字体 & UI | `ui/PixelText.ts` + `ui/Button.ts` | 数字、字母可渲染，按钮有正常/高亮/按下状态 |

### 依赖关系

```
2.1 → 2.2 → 2.3
2.1 → 2.4
2.1 → 2.5
```

### 阶段验收

- 所有核心精灵可在画面中显示
- 像素风格统一，无锯齿/模糊
- 背景三层分明，颜色正确

---

## Phase 3 - 核心玩法

**目标：** 实现游戏最核心的抓取循环

### 任务清单

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|---------|
| 3.1 | 矿工角色 | `entity/Miner.ts` | 矿工在画面顶部显示，待机动画正常 |
| 3.2 | 钩爪摆动 | `entity/Hook.ts` (摆动状态) | 钩爪以矿工为圆心左右匀速摆动 |
| 3.3 | 钩爪发射 | Hook 发射 + 延伸逻辑 | 按空格/点击后钩爪沿当前角度匀速延伸 |
| 3.4 | 碰撞检测 | `utils/collision.ts` + 碰撞判定 | 钩爪碰到矿物触发抓取，未碰到到最大距离收回 |
| 3.5 | 收回机制 | Hook 收回 + 重量影响 | 收回速度受矿物重量影响，公式正确 |
| 3.6 | 矿物生成 | `entity/Mineral.ts` + 随机布局 | 矿物按配置随机分布在地下区域 |
| 3.7 | 计分系统 | 金额累加 + 目标判定 | 抓到矿物正确加减分，关卡目标判定正确 |
| 3.8 | HUD 界面 | `ui/HUD.ts` | 实时显示时间、当前金额、目标金额 |

### 依赖关系

```
3.1 → 3.2 → 3.3 → 3.4 → 3.5
3.6 → 3.4
3.7 → 3.8
```

### 阶段验收

- 完整的抓取循环可玩：摆动 → 发射 → 碰撞 → 收回 → 计分
- 不同矿物重量影响收回速度
- HUD 信息准确实时更新
- 60 秒计时正常，时间到触发结算

---

## Phase 4 - 关卡 & 商店

**目标：** 完善游戏循环

### 任务清单

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|---------|
| 4.1 | 关卡数据配置 | `level/levels.ts` | 至少 10 关配置数据，难度递增 |
| 4.2 | 关卡管理器 | `level/LevelManager.ts` | 关卡加载、切换、进度追踪 |
| 4.3 | 结算界面 | `scene/ResultScene.ts` | 显示本关得分，达标/未达标判定，动画效果 |
| 4.4 | 道具商店 | `scene/ShopScene.ts` | 4 种道具展示、购买、金额扣除 |
| 4.5 | 道具效果系统 | 道具使用逻辑整合到 GameScene | 炸药/力量药水/幸运草/石头书效果正确 |
| 4.6 | 存档系统 | `core/Storage.ts` | 游戏进度和最高分持久化 |

### 依赖关系

```
4.1 → 4.2 → 4.3
4.4 → 4.5
4.6 (独立)
```

### 阶段验收

- 可以从第 1 关玩到最后一关
- 达标进商店→下一关，未达标→Game Over
- 道具购买和效果均正确
- 关闭页面重新打开，存档数据正确恢复

## Phase 5 - 音效 & 特效

**目标：** 增加游戏体验

### 任务清单

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|---------|
| 5.1 | 音效管理器 | `core/Audio.ts` | Web Audio API 封装，支持播放/停止/音量控制 |
| 5.2 | 游戏音效 | 音效文件 + 播放触发点 | 发射、抓取、成功、失败、倒计时音效正确触发 |

| 5.5 | 背景音乐 | 可选：8-bit 风格 BGM | 循环播放，可静音 |

### 阶段验收

- 游戏中音效反馈及时，不卡顿
- 粒子特效视觉丰富，不影响性能
- 音量可控制，支持静音

## Phase 6 - 打磨 & 发布

**目标：** 优化体验并部署

### 任务清单

| # | 任务 | 交付物 | 验收标准 |
|---|------|--------|---------|
| 6.1 | 移动端适配 | 触摸优化 + 视口处理 | 移动端流畅可玩，无操作问题 |
| 6.2 | 性能优化 | 精灵缓存全覆盖、对象池 | 全场景 60FPS，移动端 ≥ 30FPS |
| 6.3 | 游戏平衡性 | 调参后的关卡配置 | 前 3 关新手友好，后期有挑战但可完成 |
| 6.4 | 主菜单完善 | `scene/MenuScene.ts` 完整版 | Logo、开始按钮、最高分展示 |
| 6.5 | 构建部署 | Vite 生产构建 + 部署配置 | 可部署到 GitHub Pages 或静态托管 |

### 阶段验收

- 全流程可玩：菜单→游戏→商店→结算→循环/结束
- 移动端和桌面端均流畅
- 构建产物可直接部署
- 通过质量标准文档的发布前检查清单

---

## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 像素精灵制作耗时 | Phase 2 延期 | 使用 pixel-converter 工具加速；先用简单色块占位 |
| 移动端性能不足 | Phase 6 延期 | 提前在 Phase 3 测试移动端性能；预渲染缓存优先级提高 |
| 游戏平衡性问题 | 体验不佳 | Phase 3 完成后尽早进行游玩测试和调参 |
| 音效资源获取 | Phase 5 阻塞 | 使用 Web Audio API 程序生成简单音效作为备选方案 |
