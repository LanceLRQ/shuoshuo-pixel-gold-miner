# 黄金矿工 H5 - 质量标准

## 一、代码质量标准

### 1.1 TypeScript 编译

- **零 Error**：`tsc --noEmit` 必须通过，零编译错误
- **最小 Warning**：编译警告应尽量消除，无法消除的需注释说明原因
- **strict 模式**：tsconfig.json 必须开启 `strict: true`

### 1.2 ESLint 检查

- 代码提交前通过 `eslint` 检查
- 不允许使用 `@ts-ignore`，如需绕过类型检查使用 `@ts-expect-error` 并附带原因注释

### 1.3 代码复杂度控制

| 指标 | 限制 |
|------|------|
| 单函数行数 | ≤ 50 行 |
| 单文件行数 | ≤ 400 行 |
| 函数参数数量 | ≤ 5 个（超出使用配置对象） |
| 嵌套深度 | ≤ 3 层 |
| 圈复杂度 | ≤ 10 |

## 二、性能标准

### 2.1 帧率目标

| 平台 | 目标帧率 | 最低可接受 |
|------|---------|-----------|
| 桌面浏览器 | 60 FPS | 30 FPS |
| 移动端 | 60 FPS | 30 FPS |

### 2.2 性能约束

| 指标 | 阈值 |
|------|------|
| 单帧 Canvas 绑制调用 | ≤ 60,000 次 |
| 内存占用 | ≤ 50 MB |
| 首屏加载时间 | ≤ 2 秒 |
| JS Bundle 大小 | ≤ 200 KB (gzipped) |

### 2.3 性能检测方法

```typescript
// 开发环境帧率监控
if (import.meta.env.DEV) {
  // 每秒输出帧率到控制台
  // 超过阈值时输出警告
}
```

### 2.4 关键性能优化点

| 场景 | 要求 |
|------|------|
| 精灵渲染 | 必须使用离屏 Canvas 预渲染缓存，禁止每帧逐像素绘制 |
| 粒子系统 | 使用对象池，避免频繁 GC |
| 碰撞检测 | 仅检测钩爪尖端与矿物，不做全量两两碰撞 |
| 事件监听 | 使用事件委托，避免为每个元素绑定事件 |

## 三、兼容性标准

### 3.1 浏览器支持

| 浏览器 | 最低版本 |
|--------|---------|
| Chrome | 80+ |
| Firefox | 80+ |
| Safari | 14+ |
| Edge | 80+ |
| 微信内置浏览器 | 支持 |

### 3.2 设备适配

| 设备类型 | 要求 |
|---------|------|
| 桌面端 | 键盘（空格发射）+ 鼠标点击 |
| 移动端 | 触摸点击发射，竖屏优先 |
| 平板 | 触摸交互，自适应布局 |

### 3.3 屏幕适配规则

- 逻辑分辨率固定 480x640
- 等比缩放适配屏幕，保持宽高比
- 像素渲染 `imageSmoothingEnabled = false`
- 居中显示，空白区域填充背景色

## 四、游戏体验标准

### 4.1 交互响应

| 交互 | 响应时间 |
|------|---------|
| 点击/触摸到钩爪发射 | ≤ 1 帧（16ms） |
| 场景切换 | ≤ 500ms |
| 音效播放 | ≤ 50ms |

### 4.2 动画流畅度

- 钩爪摆动必须平滑无卡顿
- 矿工角色动画帧切换自然
- 粒子特效不影响主游戏帧率
- 使用 deltaTime 保证不同帧率下游戏速度一致

### 4.3 游戏平衡性

| 指标 | 标准 |
|------|------|
| 前 3 关 | 新手友好，目标金额容易达成 |
| 中期关卡 | 需要策略选择（大金块 vs 多个小金块） |
| 后期关卡 | 有挑战性但不至于不可能完成 |
| 道具系统 | 每个道具都有明确使用场景，不存在绝对最优解 |

## 五、资源质量标准

### 5.1 像素精灵

| 指标 | 标准 |
|------|------|
| 风格一致性 | 所有精灵保持统一的像素风格和配色方案 |
| 尺寸规范 | 严格遵循设计文档中的尺寸定义 |
| 透明处理 | 透明像素统一使用 `0`，无遗漏 |
| 辨识度 | 玩家能在游戏中清晰区分不同矿物类型 |

### 5.2 音效

| 指标 | 标准 |
|------|------|
| 格式 | MP3 / OGG（Web Audio API 兼容） |
| 单个文件大小 | ≤ 100 KB |
| 音效总大小 | ≤ 1 MB |
| 播放延迟 | ≤ 50ms |
